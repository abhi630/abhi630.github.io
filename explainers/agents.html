What are AI Agents?

AI that doesn't just answer — it acts. The loop, the tools, and why agents are a fundamentally different kind of system.

An AI agent is an AI that can take actions in the world — not just generate text. It perceives its environment, makes decisions, uses tools, and loops until a goal is accomplished.

The Analogy

A chatbot is like asking someone a question. An AI agent is like hiring an assistant who goes off, does the work — researching, writing, sending — and returns with the finished result.

The Key Shift

Traditional software follows fixed rules. An agent reasons about what to do next, adapts when things change, and uses judgment — not just instructions — to reach its goal.

The agent loop — perceive, plan, act, observe — is the defining pattern. Everything else (tools, memory, multi-agent systems) is built on top of this core loop.

▼
1. Perceive
The agent reads the goal, environment, any tool outputs, and prior steps — building a full picture of current state. Context accumulates with each loop. By step 3, the agent has a rich model of what's been tried, what worked, and what's still needed.
▼
2. Plan
Using reasoning, the agent decides what action to take next: which tool, what inputs, whether to decompose the task further. Planning quality determines everything. A good agent knows when to ask for clarification, when to simplify, and when a sub-agent would be faster.
▼
3. Act
The agent calls a tool or executes a step — browsing the web, writing code, reading a file, or sending a request. The action is real and consequential — a file gets written, an email gets sent, code gets executed. This distinguishes agents from chatbots.
▼
4. Observe
The result comes back. The agent reads it, updates its understanding, and decides whether the task is done or another loop is needed. Observing isn't just reading output. A skilled agent catches unexpected results, adapts its plan, and decides whether to retry or escalate.
▼
5. Done / Loop
If the goal is met, the agent reports back. If not, it loops again — re-planning with the new information from the last action. Most real tasks take 3–10 loops. Simple tasks take 1. Complex tasks (build a report) can take 20+.

Web Search

Browse live web pages, search for current information, read articles.

Why it matters: Agents can research anything in real-time, not just from training data.

Use cases:

  • Competitive research
  • Fact checking current events
  • Finding documentation

Code Execution

Write and run Python, JavaScript, shell in sandboxed environments.

Why it matters: Turns ideas into actions — analysis, file processing, calculations, automations.

Use cases:

  • Data analysis
  • Generating charts
  • Processing large files
  • Running tests

File System

Read, write, move files — PDFs, spreadsheets, images, documents.

Why it matters: Work with real documents on disk, not just pasted text.

Use cases:

  • Batch document processing
  • Report generation
  • Data extraction from PDFs

External APIs

Connect to any service with an API — Slack, email, calendar, databases.

Why it matters: An agent with API access can interact with any digital system.

Use cases:

  • Sending notifications
  • CRM updates
  • Calendar scheduling
  • Order processing

Memory / RAG

Retrieve relevant info from a knowledge base or past conversations.

Why it matters: Grounds the agent in your specific domain, not just general training.

Use cases:

  • Personalised responses
  • Company knowledge lookup
  • Past conversation recall
