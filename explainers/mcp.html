What Is MCP
(Model Context Protocol)?

The open standard that lets AI tools read live data from your apps ‚Äî without copy-pasting. How it works, why every platform is adopting it, and what it means for automation.

Every AI Tool Is an Island

Before MCP, connecting an AI to a data source ‚Äî Notion, GitHub, a database ‚Äî required a custom integration. Every tool, every connection, built from scratch. The problem wasn't the AI. It was the plumbing.

The result: AI assistants could only see what you pasted into the chat window. A 90-minute workflow became: open the tool, copy the content, switch to AI, paste it, describe the context, ask the question. Then do it again.

Key Insight

MCP is to AI tools what USB was to hardware. Before USB, every peripheral needed a proprietary connector. After USB, you plug anything into anything. MCP is building that same universal interface layer for AI and data sources.

Why It Launched in November 2024

Anthropic published the MCP specification as an open standard in November 2024. By December 2025, governance moved to the Linux Foundation ‚Äî the same body that oversees Kubernetes and Node.js. As of early 2026, there are over 10,000 active MCP servers available. OpenAI, Google, Microsoft, and AWS have all announced native support.

This is not a Claude-specific feature. It is a cross-industry standard for how AI connects to the world.

Three Moving Parts

MCP has a simple three-layer structure. Click each step to see what happens inside.

01
MCP Host ‚Äî the AI you're talking to
Claude, Cursor, Windsurf, or any MCP-compatible AI application

Role: initiates requests, surfaces results to the user

The host is the AI tool you interact with ‚Äî Claude.ai, Claude Code, a VS Code extension, or a custom app built on the API. It contains an MCP client that knows how to speak the MCP protocol. When you ask a question, the host decides which MCP servers to consult and sends requests to them. It then receives the results and folds them into its response to you.

02
MCP Protocol ‚Äî the shared language
JSON-RPC messages over stdio or HTTP/SSE ‚Äî the standard both sides speak
‚ñº
Role: defines the message format so any host can talk to any server

The protocol is the agreement. It defines exactly how a host asks a server "what can you do?" and how the server responds with a list of tools, resources, and prompts. It defines how a host says "run this function with these parameters" and how the server returns results. Because both sides follow the same spec, a server built once works with every host ‚Äî Claude, Cursor, or anything else that implements MCP.

‚Üì
03
MCP Server ‚Äî your data, made readable
Notion, GitHub, your database, your files ‚Äî each server exposes what it knows
‚ñº
Role: wraps your data source in the MCP interface

The server is a lightweight program that sits in front of a data source. It exposes three things: tools (functions the AI can call, like "search my notes" or "get file contents"), resources (data the AI can read, like the current state of a document), and prompts (pre-written instructions the server suggests for common tasks). When the host asks for something, the server fetches live data and returns it in the agreed format.

What servers expose: Tools
Functions the AI can call. Example: search_notes(query), create_issue(title, body), query_database(sql)
What servers expose: Resources
Live data the AI can read. Example: the current contents of a file, a database row, the state of a project.
What servers expose: Prompts
Suggested instructions for common workflows. The server can tell the AI "here's a good way to ask me about commits."
Where servers run
Locally on your machine (for desktop apps) or remotely via HTTP for cloud-hosted tools. Both work with the same protocol.

Key Property

Write Once, Connect Everywhere

A Notion MCP server built today works with Claude, Cursor, Windsurf, and any future MCP-compatible AI. You never rebuild the integration. The standard handles compatibility.

Key Property

Live Data, Not Snapshots

Every query fetches current data from the source. Unlike context you paste into a chat (which is immediately stale), MCP gives the AI a live window into your systems.

What This Enables

An AI that can see your Notion project notes, the open GitHub issues, and your Google Sheet of metrics ‚Äî all at once, all live ‚Äî without you doing anything except connecting the servers once. That's the practical consequence of the architecture above.

Common Questions

Things people get wrong about MCP, answered directly.

Is MCP only for Claude? +
No. MCP is an open standard published by Anthropic in November 2024 but governed independently since December 2025 by the Linux Foundation. OpenAI has added native MCP support. Google, Microsoft, and AWS have all announced compatibility. Any AI tool can implement MCP. Any server you build works across all of them.
Do I need to be a developer to use MCP? +
To use existing MCP servers ‚Äî no. Many tools like Claude.ai let you add pre-built servers through settings with no code at all. To build a custom MCP server that exposes your own data or internal tools ‚Äî yes, that requires development work. Servers can be built in Python, TypeScript, or any language with an MCP SDK available.
What's the difference between MCP and an API? +
An API defines how two systems talk to each other ‚Äî typically built for a specific integration. MCP is a meta-standard: it defines how any AI host discovers and calls any server, using a shared protocol. Think of it this way: an API is a custom-built road between two cities. MCP is the highway system ‚Äî the standard that means any vehicle (AI host) can reach any destination (server) using the same rules.
Is MCP the same as giving the AI internet access? +
Not exactly. Web browsing is one thing MCP can enable, but MCP is broader and more controlled. You choose which servers to connect. Each server only exposes what it's designed to expose. You're not giving the AI open internet access ‚Äî you're giving it structured, permissioned access to specific data sources you control. A Notion MCP server doesn't give the AI access to your email. Each connection is scoped.
What about security? Should I be concerned? +
Reasonable concern, worth taking seriously. MCP servers run with whatever permissions you grant them. A poorly written server could expose more data than intended. A compromised server is a risk. The practical steps: use well-maintained, open-source servers where you can review the code; grant minimum necessary permissions; don't connect servers to sensitive systems without understanding what they expose. The security model is like any third-party integration ‚Äî the standard itself doesn't create risk, implementation does.
How is MCP different from Agent Skills? +
MCP and Agent Skills solve adjacent but different problems. MCP is about data access ‚Äî giving the AI a live connection to external systems. Agent Skills are about workflow knowledge ‚Äî teaching the AI how to perform multi-step tasks using a set of tools. MCP is the infrastructure layer. Skills sit on top of it. A Skill might use an MCP server to read your Notion data, then use another to write results back to a database. The next post in this series covers exactly when to use which.
